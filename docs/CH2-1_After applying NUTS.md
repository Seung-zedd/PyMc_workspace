## 1. 사후분포 요약 통계량 (Summary Table)

가상 데이터의 생성 규칙인 $y = 2.0x + 1.0 + \text{오차}(1.5)$를 바탕으로, 고성능 NUTS 엔진이 추정한 모수 결과입니다.

| 모수 (Parameter)          | 실제 정답 (True) | 추정 평균 (`mean`) | 94% HDI 구간 (`hdi_3% ~ hdi_97%`) |
| :------------------------ | :--------------: | :----------------: | :-------------------------------: |
| **`w` (기울기)**          |      `2.0`       |    **`2.021`**     |          `1.929 ~ 2.110`          |
| **`b` (절편)**            |      `1.0`       |    **`0.739`**     |          `0.236 ~ 1.268`          |
| **`sigma` (노이즈 크기)** |      `1.5`       |    **`1.384`**     |          `1.213 ~ 1.579`          |

---

## 2. 핵심 지표 해석

- **`w` (기울기):** 무작위 노이즈가 섞인 환경에서도 실제 인과관계 가중치(`2.0`)를 `2.021`로 매우 정밀하게 복원해 냈습니다.
- **`b` (절편):** 데이터 생성 시 발생한 무작위 노이즈의 쏠림으로 인해 추정 평균(`0.739`)은 다소 낮게 계산되었으나, 94% HDI 구간(`0.236 ~ 1.268`) 안에 실제 정답(`1.0`)이 안정적으로 포함되어 있으므로 통계적으로 유효한 추론입니다.
- **`sigma` (데이터 노이즈):** 선형 회귀 직선 주변의 오차 크기를 `1.384`로 추정하며, 실제 노이즈 설정값(`1.5`)에 근접한 결과를 도출했습니다.

## 4. MCMC 엔진 비교 분석 (Metropolis vs NUTS)

실무 프로젝트에서 어떤 엔진을 선택해야 하는지에 대한 가이드라인입니다.

| 비교 항목          | Metropolis (MH)                               | NUTS (Hamiltonian MC)                   |
| :----------------- | :-------------------------------------------- | :-------------------------------------- |
| **작동 원리**      | 무작위 걷기 (Random Walk)                     | 확률 경사면 탐색 (Gradient-based)       |
| **추천 변수 개수** | 1~3개 (단순 모델)                             | **제한 없음 (복잡한 실무 모델)**        |
| **수렴 속도**      | 매우 느림 (제자리걸음 많음)                   | 매우 빠름 (목표 지향적 탐색)            |
| **사용 편의성**    | Step Size를 수동 설정해야 함                  | **자동 최적화 (Tuning)**                |
| **주요 용도**      | 교육용, 이산형(Discrete) 변수                 | **실무 베이지안 모델링 표준**           |
| **치명적 단점**    | 차원의 저주에 약함 (변수 증가 시 신뢰도 급락) | 수식의 미분이 필요함 (PyMC가 자동 수행) |

### 💡 실무 적용 전략

1.  **변수가 2개 이상이면 무조건 NUTS가 기본입니다.** `pm.sample()`에서 별도의 `step` 인자를 주지 않으면 PyMC가 자동으로 NUTS를 선택합니다.
2.  **Metropolis는 언제 쓰나요?**
    - 파라미터가 정수(Count 등)인 **이산형 데이터**일 때 (NUTS는 미분이 불가능한 이산형 변수에 쓰지 못함)
    - 모델의 구조가 너무 단순해서 NUTS를 돌리는 오버헤드가 더 클 때 (극히 드묾)
3.  **결론:** 실무에서는 **"NUTS를 기본으로 사용하되, Divergence(발산) 경고가 뜨면 모델의 구조를 수정한다"**는 공식으로 접근하는 것이 가장 신뢰성 있는 모델을 구축하는 방법입니다.

## 5. Known Issues (알려진 문제)

- **현상:** 실행 시 터미널에 `Warning: Could not load "...\gvplugin_pango.dll"...` 등의 Graphviz 관련 경고가 다수 출력됨.
- **원인:** Windows 환경의 Conda 시스템과 Graphviz 패키지 간의 폰트 렌더링(DLL) 의존성 충돌 문제.
- **조치 사항:** 해당 경고는 모델의 내부 구조를 이미지(`model_structure.png`)로 출력할 때만 발생하는 시각화 관련 버그입니다. MCMC 샘플링 성능이나 베이지안 연산 결과에는 **단 0.1%의 영향도 주지 않으므로 실무에서는 무시(Ignore)하고 진행해도 무방합니다.**
